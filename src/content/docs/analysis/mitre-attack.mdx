## Introduction

The MITRE ATT&CK framework provides a standardized taxonomy for adversary behavior. This page maps the techniques observed across all specimen families in the collection, demonstrating how attack patterns evolved from simple bot commands in 2002 to sophisticated multi-stage operations by 2009.

Each entry includes the ATT&CK technique ID, name, implementing specimens, and a brief description of how the technique manifests in the source code.

| ID | Technique | Specimens | Description |
|----|-----------|-----------|-------------|
| **T1592** | Gather Victim Host Information | IcePack, Fragus, Fiesta3 | Exploit kits fingerprint victim browser, OS, installed plugins, and geographic location via GeoIP database before selecting exploits |
| **T1590** | Gather Victim Network Information | hydra-2008.1, rBot, 120-Series | Network scanning modules probe IP ranges for open ports and vulnerable services; Hydra scans `/16` ranges for Telnet/HTTP |

## Evolutionary Observations

### Technique Accumulation Over Time

| Era | Typical Technique Count | New Tactics Added |
|-----|------------------------|-------------------|
| **2002 (sdBot)** | ~5 | Basic: C2, persistence, DoS, download |
| **2003 (SpyBot)** | ~10 | + Credential access, lateral movement (IM/P2P) |
| **2004 (rxBot)** | ~20 | + Defense evasion, discovery, multiple exploits |
| **2005 (NZM)** | ~25 | + Organized evasion, more exploit modules |
| **2007 (120-Series)** | ~30 | + Advanced evasion (encryption, bot-killer, SP2 patching) |
| **2007–2009 (Exploit Kits)** | ~10 (specialized) | + Reconnaissance, execution guardrails, client-side exploitation |
| **2009 (Zeus)** | ~25 (specialized) | + MitB, data manipulation, structured exfiltration, impact commands |

### Key Patterns

1. **Defense evasion techniques grew fastest** - From zero techniques in sdBot to 6+ in the 120-Series, reflecting the arms race with AV vendors
2. **Credential access diversified** - From none (sdBot) to keylogging (SpyBot) to multi-source harvesting (rxBot) to real-time browser hooking (Zeus)
3. **C2 evolved from simple to resilient** - Single IRC server → backup servers → encrypted IRC → P2P mesh → HTTP polling
4. **Initial access shifted** - From network exploits (server-side, wormable) to client-side exploitation (drive-by downloads via exploit kits)
5. **Collection became structured** - From dumping text to IRC channels to searchable MySQL databases with typed data categories
